# 적절한 서브에이전트를 사용하여 TodoList 앱의 PRD(제품 요구사항 문서)를 마크다운 형식으로 작성해서 docs/1-prd.md로 저장해줘.

- 앱 이름: TodoList 앱
- 앱 목적: 개인 할 일을 등록·완료·삭제할 수 있는 간단한 웹 앱
- 타깃 사용자: 혼자 할 일을 관리하고 싶은 1인 개발자 또는 개인 사용자
- 핵심 기능: ① 회원가입 ② 로그인 ③ 할 일 추가 ④ 완료 체크 ⑤ 할 일 삭제 ⑥ 전체 목록 조회
- 기술 스택:
 * 프론트엔드: React + Vite + TypeScript (Vercel 배포)
 * 백엔드: Node.js + Express + JavaScript + pg 라이브러리 (Vercel 배포)
 * DB: Supabase PostgreSQL (DB 기능만 사용, Auth 등 타 기능 미사용)
 * 인증: 백엔드 자체 JWT 인증 (Supabase Auth 미사용)
 * 제외 기능(out of scope): 팀 공유, 알림, 모바일 앱

# PRD에는 다음 섹션을 포함해줘:
1. 개요 (목적 & 배경)
2. 타깃 사용자
3. 사용자 스토리
- "나는 [사용자]로서 [목표]를 위해 [기능]을 원한다" 형식으로 핵심 기능별 작성
- 각 스토리마다 수락 기준(Acceptance Criteria) 2~3개 포함
4. 핵심 기능 목록 (우선순위: 높음/중간/낮음 표시)
5. 비기능 요구사항 (다음 항목을 반드시 포함)
- 반응형 웹 UI: 모바일(360px 이상)·태블릿·데스크톱 모든 해상도 지원
- TailwindCSS sm/md/lg 브레이크포인트 기준으로 레이아웃 전환
- 성능, 보안, 접근성 등 기타 비기능 요구사항
6. 범위 외(out of scope)
7. 용어 정의 (필요시)
